# 配件素材图 QA 与颜色策略 v1

Last reviewed: 2026-05-29 Asia/Shanghai

Context note: asset/color-policy subsystem reference only. Do not read during normal UI/account handoff.

目的：减少模型把“配件参考图”误当成完整场景、整车风格或颜色指令，避免出现换背景、换车型、未选轮毂被改、车身覆盖件颜色冲突等 bad case。

## 1. 核心结论

当前 bad case 不能只靠 Prompt 修。配件参考图本身也会强烈影响模型：

- 如果参考图带背景、带其它车辆、带展台、带文字、水印或复杂角度，模型容易把这些内容带进结果。
- 如果参考图颜色与用户配置冲突，例如黑色碳纤维机盖 + 用户选择纳多灰整车，模型容易直接继承参考图黑色。
- 如果一次上传过多配件参考图，模型容易把“未选择的类别”也一起泛化生成。
- 对轮毂、机盖、前唇、尾翼这类视觉强特征配件，参考图质量比 Prompt 更关键。

## 2. 配件图素材标准

### 展示图与生图图分离

后台和前台展示不应该等同于生图入参。

- `displayImage`: 只用于后台/前台展示，1 张即可，要求好看、清楚、容易被运营识别。
- `generationReferences`: 只用于喂给生图模型，可以是同一配件的多张图，要求角度、安装关系和材质信息完整。

也就是说：用户界面展示一张配件主图就够了，但模型入参可以按需上传同一配件的多张参考图。尤其是机盖、尾翼、前唇、侧裙、扩散器、排气这类依赖安装角度的配件，单张产品图通常不够。

### 必须满足

- 单一配件为主体，不要整车图。
- 背景尽量纯白、透明、浅灰或干净展台。
- 无水印、无文字、无包装、无人物、无其它车辆。
- 配件边缘完整，不能被裁切。
- 最好 1024px 以上，最低不低于 768px。
- 角度与安装视角接近真实使用位置。
- 同一个配件保留 1 张展示主图；生图参考图建议 2-4 张，按配件复杂度决定。

### 不建议使用

- 社交媒体截图。
- 只有远景、强滤镜、强背景的安装效果图。
- 带强烈摄影背景的广告图。
- 角度过于艺术化或局部特写过重的图。
- 低清、小图、压缩严重图片。
- 多个配件混在一张图里。
- 颜色与配置强冲突但没有 metadata 说明的图。

注意：安装在同车型或相近车型上的清晰实拍图，不应该一律排除。对于机盖、尾翼、包围、排气等需要安装关系的配件，清晰安装图比单独产品图更有价值，但必须标记为 `install_context`，并在 Prompt 里声明只参考安装位置和配件形状，不继承背景、车身颜色、轮毂或场景。

## 2.1 不同配件建议上传什么图

| 配件类型 | 展示主图 | 生图参考图建议 | 说明 |
|---|---|---|---|
| 轮毂 | 正侧白底产品图 | 正侧产品图 + 车上安装图可选 | 正侧图最重要；车上图只辅助理解 fitment |
| 卡钳 | 单个卡钳产品图 | 卡钳产品图 + 轮毂内安装位置图 | 需要避免模型改轮毂 |
| 机盖 | 干净产品图或安装图 | 产品图 + 同车型/相近车型安装图 + 局部轮廓图 | 单独产品近景不够，容易导致轮廓错位 |
| 前唇 | 产品图 | 正前/前 45 度安装图 + 产品图 | 需要明确安装在前保险杠下沿 |
| 侧裙 | 产品图 | 侧面安装图 + 前/后端局部图 | 单独长条产品图很难表达安装位置 |
| 尾翼/后扰流 | 独立尾翼产品图 | 产品图 + 后视/后 45 度安装图 | 45 度原车图可评低调尾翼的部分可见边缘 |
| 扩散器 | 产品图 | 后视安装图 + 产品图 | 只适合后视/后 45 度原车图 |
| 排气 | 尾嘴产品图 | 后视安装图 + 尾嘴局部图 | 只在排气可见的原车图上传 |
| 尾灯/灯罩 | 单侧或一对尾灯产品图 | 正后安装图 + 产品图 | 必须保持原尾灯外轮廓和位置 |
| 格栅 | 产品图 | 产品图 + 车头安装图可选 | 车头正面或前 45 度有效 |
| 车身颜色/贴膜 | 色卡/材质示意可展示 | 通常不上传图片，使用颜色名、HEX、材质文本 | 小色卡图容易污染画面 |

## 2.2 generationReferences 字段建议

```json
{
  "displayImage": "parts/hood/display.jpg",
  "generationReferences": [
    {
      "url": "parts/hood/product-clean.jpg",
      "role": "shape_reference",
      "view": "product",
      "priority": 1,
      "useForViews": ["front", "front_three_quarter"],
      "promptHint": "只参考机盖外形、轮廓和折线，不继承黑色碳纤维颜色"
    },
    {
      "url": "parts/hood/installed-f82-front-45.jpg",
      "role": "install_context",
      "view": "front_three_quarter",
      "priority": 2,
      "useForViews": ["front_three_quarter"],
      "promptHint": "只参考安装边界、缝隙和与灯眉/前挡风玻璃的关系，不继承背景和车身颜色"
    }
  ]
}
```

### reference role

- `shape_reference`: 配件形状、轮廓、折线。
- `material_reference`: 碳纤维、亮黑、金属、磨砂等材质。
- `color_reference`: 明确要求继承的颜色。
- `install_context`: 配件装到车上的位置、比例、透视、缝隙、遮挡关系。
- `avoid_upload`: 素材只展示，不上传给生图模型。

## 3. 当前素材风险初评

| 类别 | 当前素材 | 风险 | 建议 |
|---|---|---|---|
| 轮毂 | BBS LM-R 系列 320px 图片 | 中 | 可用于测试，但尺寸偏小；建议补 1024px 白底产品图 |
| 卡钳 | Brembo / AP Racing | 中 | 适合识别颜色和结构；最好补透明/白底图 |
| 前唇 | APR 前唇 webp | 高 | 形状细长，容易被模型忽略或误贴；建议补安装角度图 + 白底图 |
| 侧裙 | RSC 侧裙 webp | 高 | 侧裙很难从单独素材理解安装位置；建议补侧视安装示意图 |
| 机盖 | Seibon 黑色碳纤维机盖 | 高 | 与整车改色冲突；必须加 colorPolicy，不要让模型直接继承黑色 |
| 格栅 | BMW 黑色格栅 | 中 | 可用，但需要纯产品图，不要带车头背景 |
| 尾翼 | RSC 后扰流 | 中 | 小尾翼容易被生成成大 GT 翼；Prompt 需明确低矮鸭尾 |
| 扩散器 | RSC 扩散器 | 高 | 只在尾部视角有效；前 3/4 原图时不应强行出现 |
| 排气 | Akrapovic 尾嘴 | 高 | 只在尾部视角有效；前脸/侧前图不应强行出现 |
| 尾灯 | VLAND 尾灯 | 高 | 只在尾部视角有效；前脸图不应生成 |
| 后视镜 | APR 碳纤维后视镜 | 中 | 小件容易被忽略；需要局部位置提示 |
| 贴膜 | 3M 小色卡 avif | 高 | 1.4KB 太小，不适合作为视觉参考；应改为色值/材质文本，不上传图片 |

## 4. 新增后台字段建议

### part_assets 增加字段

```json
{
  "displayImage": "string",
  "generationReferences": [],
  "referenceQuality": "high | medium | low",
  "referenceRole": "shape_only | material_only | color_only | install_position | full_part_reference",
  "paintBehavior": "body_color | exposed_carbon | part_reference_color | keep_original",
  "viewApplicability": ["front", "front_three_quarter", "side", "rear_three_quarter", "rear"],
  "uploadToImageModel": true,
  "negativeHint": "不要继承参考图背景、文字、水印、其它车辆"
}
```

### 字段解释

- `referenceQuality`: 素材是否适合上传给生图模型。
- `referenceRole`: 告诉模型这张图只用于什么，例如只参考形状，不继承颜色。
- `displayImage`: 展示主图，只服务后台/前台选择，不一定上传给模型。
- `generationReferences`: 生图参考图组，可包含产品图、安装图、材质图、局部图。
- `paintBehavior`: 解决机盖、包围、碳纤维件与整车改色冲突。
- `viewApplicability`: 原图视角不匹配时不要上传或不要强制生成。
- `uploadToImageModel`: 低质量素材可只走文本 Prompt，不上传图片。

## 5. 颜色策略

### 默认规则

| 配件类型 | 默认 paintBehavior | 说明 |
|---|---|---|
| 机盖 | body_color | 用户选择整车颜色时，机盖默认车身同色 |
| 前/后保险杠主体 | body_color | 跟随车身颜色 |
| 侧裙主体 | body_color | 跟随车身颜色 |
| 前唇/侧裙延伸/扩散器 | exposed_carbon | 可保持亮黑碳纤维 |
| 格栅 | part_reference_color | 保持亮黑 |
| 轮毂 | part_reference_color | 只有选中轮毂时才改 |
| 卡钳 | part_reference_color | 只改卡钳颜色，不改轮毂 |
| 后视镜壳 | exposed_carbon 或 body_color | 以后在后台配置 |
| 贴膜/车身颜色 | body_color | 不上传色卡小图，直接用颜色文本 |

### 机盖冲突规则

如果同时满足：

- 用户选择了整车颜色，例如纳多灰；
- 用户选择了机盖；
- 机盖素材是黑色碳纤维；
- 后台没有明确配置“裸露碳纤维机盖”；

则最终 Prompt 必须写：

```text
机盖只参考 OE 轮廓、缝隙位置和安装形状，不继承参考图的黑色碳纤维颜色。由于用户选择了整车纳多灰，机盖必须渲染为纳多灰车身同色，保持真实漆面反光和原厂机盖缝隙。
```

## 6. 参考图上传策略

### 推荐

先做三组跑批：

| 组 | 上传内容 | 目的 |
|---|---|---|
| Text Only | 只上传原车图，不上传配件图，只用文字描述配件 | 判断 Prompt 本身能否保持背景和未选项 |
| High Quality Only | 原车图 + 质量高的单件产品图 | 判断参考图是否提升配件准确率 |
| All Current Assets | 原车图 + 当前所有选中配件图 | 判断当前素材是否引入干扰 |

如果 Text Only 保真明显更高，而 All Current Assets bad case 增多，说明素材图需要替换或降级为文本参考。

### 不要一次上传太多低质量图

一张原图 + 5 张以上配件图，会显著提高模型“重构整车”的概率。配置模式建议：

- 1-2 个配件：可上传配件图。
- 3-5 个配件：只上传高质量、关键形状配件图，其它用文本。
- 6 个以上配件：优先文本描述 + 分阶段生成，不建议全部上传。

## 7. Prompt 需要新增的素材约束

```text
参考配件图的优先级低于原车图和后台配置。
原车图是唯一画布，配件图不是画布。
配件图只用于理解配件的局部形状、材质和安装位置，不继承配件图的背景、光线、车身、轮毂、颜色或场景。
如果配件图颜色与后台颜色配置冲突，以后台颜色配置为准。
如果配件图包含整车或其它车辆，只提取目标配件，不要复制其它车辆的任何部分。
如果某张配件图质量低、角度不适合或与当前视角不匹配，宁可忽略该参考图，也不要改变原车、背景或未选类别。
```

## 8. 后续资产替换优先级

1. 机盖：补“白底 OE 形状图”和“同车型/相近车型安装实拍图”，不要只用黑色碳纤维产品近景图。
2. 前唇/侧裙：补车头/侧面安装角度图，但要裁掉其它车辆背景。
3. 轮毂：补 1024px 正侧白底产品图。
4. 贴膜/颜色：不要用小色卡图片，改用颜色名称、HEX、材质描述。
5. 尾部件：只在后视角原图测试，不要在前 3/4 图强行生成。
