# Archived Mobile Design Notes

Last condensed: 2026-05-29 Asia/Shanghai

These notes are historical inspiration for the early mobile redesign. They are not current source of truth.

Durable direction that still applies:

- mobile should feel app-like, dark, compact, image-first, and operation-focused
- desktop UI should not be changed by mobile polish
- auth and subscription should use full-screen mobile surfaces, not cramped centered modals
- bottom sheets and thumb-reachable controls fit the mobile workflow

Obsolete or superseded:

- public mobile theme switching is currently hidden
- old dock proposals are superseded by current Parts/Paint/Height/Details behavior
- current access/quota banner, profile pages, and message center are implemented in the Next.js app, not this prototype

Use current app files for implementation details.
